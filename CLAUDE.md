## コーディング規約

```cpp
// メンバ変数
m_variableName

// クラス名
PascalCase

// 関数名
camelCase

// 定数
UPPER_SNAKE_CASE
```

- ヘッダに実装を書かない（テンプレートを除く）
- 例外は使わない（エラーはリターンコードかオプション型）
- `std::vector` を基本コンテナとして使う

---
## ポインタ・メモリ管理
- 原則: `std::unique_ptr` を使用。所有権の移動は `std::move`。
- 共有: `std::shared_ptr` は、PhysXのオブジェクトなど共有が必須な場合のみ。
- 借用: `T&` (参照) を最優先。`nullptr` を許容する場合のみ生ポインタ `T*` を使用。
- 生ポインタの用途:
  - 所有権に関与しない一時的なアクセス
  - C API (OpenGL, PhysX等) への引数
  - nullptr 許容のオプション参照
